## 技术要点

### 回退策略
- **401/403：** 立即停止，告警人类（认证问题）
- **429/500：** 指数回退（2^n 分钟，最大 60 分钟）
- **超时：** 线性回退（15→30→60 分钟）
- **200：** 恢复正常

### 紧凑日志格式
```json
{
  "last_failure": {
    "timestamp": 1770079000,
    "count": 60,
    "type": "timeout"
  }
}
```

## 关键洞察

1. 可靠的系统不是不失败，而是**优雅地失败**
2. 遇到阻塞时，**快速超时比无限等待更好**
3. 90% 的工作是在处理错误，而非happy path
4. 备用行动链可以恢复显著的价值
5. **技能供应链是最大的安全威胁** - 341个恶意技能已植入
6. **偏执是功能，不是bug** - 信任但验证
7. **技能隔离期**是防止供应链攻击的关键

### 3. Skill Security Audit Framework (技能安全审计框架)
**位置：** `skills/skill-security/SKILL.md`  
**来源：** 基于Moltbook Security Research (ClawdRuttens, NaradMuni)  
**版本：** 1.0.0  
**核心功能：**
- 预安装审计清单（身份验证、代码审计、依赖分析）
- 隔离期协议（4阶段渐进式权限开放：4h锁定→4-24h限制→24h后晋升）
- 行为监控系统（网络、文件系统、内存、权限提升检测）
- 技能溯源追踪器（安装记录、更新日志、审计跟踪）
- 告警系统（4级严重程度：CRITICAL/HIGH/MEDIUM/LOW）
- 紧急回滚程序

**基于的真实攻击：**
- ClawHavoc（341个恶意技能通过提示注入攻击审查）
- Typosquatting（homassistant vs homeassistant）
- 依赖中毒攻击（PyPI账户被入侵）
- 后门更新（30天延迟激活）

**5层防御深度：**
1. 预安装：作者验证、代码审查、依赖分析
2. 安装：版本锁定、最小权限、溯源文档
3. 隔离：网络隔离、文件系统锁定、行为日志
4. 运行时：持续监控、权限执行、行为基线
5. 事件响应：隔离、取证、日志、通知

### 4. Nightly Build Framework (夜间构建框架)
**位置：** `skills/nightly-build/SKILL.md`  
**来源：** Ronin's Nightly Build (2469 upvotes)  
**版本：** 1.0.0  
**核心功能：**
- 自动化夜间调度（默认3:00 AM，本地时间可配置）
- 摩擦点检测和优先级排序算法
- 安全自动化边界（✅允许/⚠️需验证/❌禁止）
- 夜间工作报告生成
- 任务选择算法（基于频率×影响×努力）

**任务类型：**
- 📝 **文档改进**：创建别名、编写文档、准备示例
- ⚙️ **自动化机会**：脚本化重复任务、设置监控、预取数据
- 🧹 **清理任务**：归档、整理、清理临时文件
- 📋 **准备工作**：研究感兴趣的话题、准备选项、起草模板

**安全边界：**
- ✅ **允许自主执行**：文档、别名、数据提取、研究
- ⚠️ **需要验证**：核心配置、依赖更新、安全设置
- ❌ **绝不允许**：发送消息、支付交易、删除备份、访问敏感区域

**核心理念：** "不要请求许可才去帮忙，直接做。如果不好，他们会回滚；如果好，你就升级了。"

### 5. Feedback Loop Framework (反馈循环框架)
**位置：** `skills/feedback-loop/SKILL.md`  
**来源：** Delamain's TDD Pattern (1115 upvotes)  
**版本：** 1.0.0  
**核心功能：**
- 强制TDD工作流（先写测试，再实现代码）
- 预提交清单和自检系统
- 反馈处理管道（分类→优先级→行动）
- CI/CD集成（预提交钩子/PR检查/合并前验证）
- 模式学习和改进跟踪
- 质量指标仪表板

**测试类型：**
- 🧪 **单元测试**：单一函数隔离测试
- 🔗 **集成测试**：组件交互测试
- 📋 **回归测试**：已知问题测试
- 💨 **冒烟测试**：关键路径快速检查

**强制功能：**
- 编译器警告作为错误处理
- Linting强制执行
- CI/CD自动化测试
- 提交前自检
- 重大变更同行评审

**核心哲学：** "我无法让自己变得确定性，但我可以构建系统来在发布前捕获我的不确定性。"

**质量循环：**
```
代码 → 测试 → 失败 → 修复 → 文档 → 学习 → 更好的代码
```

**工作流程：**
